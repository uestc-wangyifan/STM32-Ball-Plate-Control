# 联调验证清单 (上机前必须执行)

## 1. 编译检查
*   **操作**：在 Keil/STM32CubeIDE 中全量重新编译 `main.c` 和整个工程。
*   **预期现象**：0 Error，0 Warning（除原有库相关的隐式警告外，不要有 `math.h` 或函数未声明的新警告）。
*   **异常说明**：如果出现 `isfinite` 未定义报错，说明当前工具链未开启 C99 或 Newlib 支持，需要补一个宏 `#define isfinite(x) (!isnan(x) && !isinf(x))`。

## 2. 串口/摄像头启动检查
*   **操作**：STM32 上电，插上串口，PC 端执行 `python vision.py --port COMx`（将 COMx 替换为实际端口号）。不要将球放在相机视野内。
*   **预期现象**：
    *   OLED 显示屏闪过 "System Booting..." 后，停在状态显示界面 "Stat: WAIT VIS"（或类似初始化状态）。
    *   舵机在 500ms 后动作，并且固定在中心水平位置（不因环境发抖）。
    *   PC 端成功打开摄像头及串口，终端打印 `[INFO] 摄像头分辨率: 640x480`。
*   **异常说明**：如果 MCU 报错卡死（如停在 `Error_Handler`，OLED 卡住），说明串口 DMA、定时器启动失败，请检查连线。如果舵机疯狂抖动，说明干扰过大或 PWM 未能固定在 1500。

## 3. LOST/恢复检查
*   **操作**：
    1. PC 端启动后，用手在摄像头前挥动一个色球让视觉识别到跳变的坐标，然后再把球拿走。
    2. 等待超过 120ms (`RX_TIMEOUT_MS`)。
*   **预期现象**：
    *   识别到球瞬间：OLED 显示 "Stat: TRACKING"，舵机随目标尝试跟随。
    *   拿走球之后：PC 端打印 / 界面显示 `LOST` 持续计数发送。
    *   OLED 显示 "Stat: LOST/CENT"，**舵机必须立即水平归中**。
*   **异常说明**：如果球拿走后舵机卡在倾斜位置不回来，说明 `Control_EnterLostState()` 没有被正确调用或归中逻辑未生效。

## 4. PID 热调参检查
*   **操作**：使用串口助手（或者在 Python 脚本对应逻辑里），向运行中的 STM32 手工发送两帧数据：
    1. 合法帧：`PID:1.5,0.05,0.3`（末尾可带换行/空格）。
    2. 非法脏帧：`PID:1.5,0.05,0.3abc` 或 `PID:1.5,0.05`。
*   **预期现象**：
    *   合法帧发送后：电机动作平滑过渡，不产生剧烈跳动（由于改写了 `pid_y = pid_x` 且有 `PID_Reset`，修改参数不应该引发轴无端猛偏）。
    *   脏帧发送后：STM32 继续保持原有稳态，不更新新参数（可以故意发个极端值如 `PID:9.0,0,0` 来鉴别脏帧是否被误执行）。
*   **异常说明**：如果合法帧发出瞬间某单轴甚至双轴狂抽搐，说明清理历史状态时引起了计算跳变。如果手感因脏帧改变，说明解析校验失败。

## 5. 分辨率/标定兼容检查
*   **操作**：
    1. 确保目录下存在修改前的老版本 `config.json`（无 `calibration_resolution` 字段）。
    2. 运行 `python vision.py`。
    3. 标定后，使用按键 `s` 保存新的配置，关闭程序再重启。
*   **预期现象**：
    *   第一次加载旧配置文件，应当打印：`[WARN] 标定分辨率 ... 与当前 ... 不一致，已强制重新标定`，并且弹出要求用户手动点击四个角的标定流程。
    *   保存后第二轮重启，应当打印 `[INFO] 已加载标定点（按 'r' 重新标定）`，正常复用。
*   **异常说明**：如果强制标定没有触发，说明 JSON 字段容错逻辑执行异常，后续可能会因为分辨率变动带来极大的映射偏差。

## 6. 实球上板前空载检查
*   **操作**：PC 端识别到色球，你用手在摄像头视野范围内**缓慢、平滑地**移动球的 XY 坐标（此时**球不需要放置在板子上**，手持悬空移动即可）。
*   **预期现象**：
    *   观察电机输出杆：电机应当有**平滑、单调**的往复动作（得益于低通系数 0.5 更快的响应，以及首帧直接 seed 消除抖动爬升）。
    *   **死区测试**：如果在目标中心 ±2mm 内（在手不抖的情况下）细微挪动，电机绝对不产生跟随。
*   **异常说明**：如平滑游走时电机偶发断层式的生硬反抽，说明存在串口被截断或丢帧干扰（尚未修复的 `T1` 短帧/残帧漏洞），这是一个极强的风险信号。
